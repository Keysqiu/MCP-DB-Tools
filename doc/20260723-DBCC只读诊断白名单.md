# DBCC 只读诊断子命令白名单

## 目标

放宽 [SqlGuard](../src/McpDbTools.Server/Security/SqlGuard.cs) 对 SqlServer `DBCC` 的拦截，允许指定的**只读诊断子命令**通过；写/破坏类子命令继续拒。

现状：SqlServer 白名单 `SELECT/WITH/EXEC/EXECUTE/SP_*`，`DBCC` 不在表里，首关键字校验直接 Deny。运维/排障常需 `DBCC SQLPERF(logspace)`、`DBCC SHOW_STATISTICS`、`DBCC CHECKDB` 等只读诊断，当前全拒。

## 范围

- [src/McpDbTools.Server/Security/SqlGuard.cs](../src/McpDbTools.Server/Security/SqlGuard.cs) — 新增 DBCC 子命令白名单与校验分支
- [src/McpDbTools.Tests/SqlGuardTests.cs](../src/McpDbTools.Tests/SqlGuardTests.cs) — 补 DBCC 用例
- 仅 SqlServer；MySQL/Oracle 不动（无 DBCC 等价物）

## 风险评估

DBCC 子命令集庞大，混合读写。整体放行 DBCC = 放行所有子命令，**不可接受**。必须做子命令二次校验。

### 放行的只读子命令白名单

| 子命令 | 用途 | 备注 |
|---|---|---|
| `SHOW_STATISTICS` | 索引/列统计直方图 | 只读 |
| `SHOWCONTIG` | 索引碎片 | 只读（含 WITH FAST 等只读选项） |
| `CHECKDB` | 整库一致性检查 | 基础形态只读；**带 REPAIR_* 拒** |
| `CHECKTABLE` | 单表一致性检查 | 同上 |
| `CHECKALLOC` | 分配一致性 | 同上 |
| `CHECKCATALOG` | 目录一致性 | 只读 |
| `SQLPERF` | 如 `SQLPERF(logspace)` 事务日志空间 | 仅 logspace 用法，只读 |
| `INPUTBUFFER` | 会话最近语句 | 只读 |
| `OPENTRAN` | 最旧活动事务 | 只读 |
| `USEROPTIONS` | 当前 SET 选项 | 只读 |
| `TRACESTATUS` | 显示 trace flag 状态 | 只读（`TRACEON/TRACEOFF` 改实例，**不入白名单**） |
| `PROCCACHE` | 计划缓存汇总 | 只读（`FREEPROCCACHE` 清缓存，**不入白名单**） |

### 显式拒的写/破坏类（不在白名单即拒，无需单独枚举）

- `DBCC WRITEPAGE` — 直接写 mdf 页字节，可破坏库
- `DBCC SHRINKDATABASE` / `SHRINKFILE` — 收缩数据/日志
- `DBCC DBREPAIR` — detach
- `DBCC TRACEON` / `TRACEOFF` — 改实例全局开关
- `DBCC FREEPROCCACHE` / `DROPCLEANBUFFERS` — 清缓存
- `DBCC CHECKDB WITH REPAIR_ALLOW_DATA_LOSS` / `REPAIR_REBUILD` / `REPAIR_FAST` — 修复写

### 修复选项二次拦截

`CHECKDB/CHECKTABLE/CHECKALLOC` 基础形态只读，但 `WITH REPAIR_*` 会写数据/破坏。即使子命令在白名单，检测到下列修饰符仍 Deny：

- `REPAIR_ALLOW_DATA_LOSS`
- `REPAIR_REBUILD`
- `REPAIR_FAST`
- `REPAIR`（保守起见作为独立词拒，避免遗漏变体）

### 其他风险

- **黑名单仍全程生效**：DBCC 子命令过白名单后，整段 SQL 仍走 [SqlGuard.cs:101-115](../src/McpDbTools.Server/Security/SqlGuard.cs#L101-L115) 三层合并黑名单，多层兜底。
- **多语句注入**：当前实现 [SqlGuard.cs:88](../src/McpDbTools.Server/Security/SqlGuard.cs#L88) `Split(';')[0]` 只取首句判断，黑名单扫整段。DBCC 作为首句过白名单，第二句写操作靠黑名单兜底；若黑名单未配 DROP/DELETE 等会漏 → 依赖三层合并黑名单完整性。本次不改此结构。
- **首关键字大小写**：normalized 已 ToUpperInvariant，子命令提取后用 `OrdinalIgnoreCase` 比较。

## 实施步骤

### 1. SqlGuard 新增 DBCC 白名单常量

```csharp
/// <summary>SqlServer DBCC 只读诊断子命令白名单（仅基础只读形态，修复选项单独拦截）。</summary>
private static readonly IReadOnlySet<string> SqlServerDbccReadOnlySubcommands =
    new HashSet<string>(StringComparer.OrdinalIgnoreCase)
    {
        "SHOW_STATISTICS", "SHOWCONTIG",
        "CHECKDB", "CHECKTABLE", "CHECKALLOC", "CHECKCATALOG",
        "SQLPERF", "INPUTBUFFER", "OPENTRAN", "USEROPTIONS",
        "TRACESTATUS", "PROCCACHE"
    };

/// <summary>DBCC 修复选项修饰符（即使子命令在白名单也拒）。</summary>
private static readonly IReadOnlySet<string> DbccRepairModifiers =
    new HashSet<string>(StringComparer.OrdinalIgnoreCase)
    {
        "REPAIR_ALLOW_DATA_LOSS", "REPAIR_REBUILD", "REPAIR_FAST", "REPAIR"
    };
```

[SqlGuard.cs:53-57](../src/McpDbTools.Server/Security/SqlGuard.cs#L53-L57) SqlServer 集合追加 `"DBCC"`。

### 3. 白名单校验后插入 DBCC 子命令分流

现有首关键字白名单判断（[SqlGuard.cs:91-97](../src/McpDbTools.Server/Security/SqlGuard.cs#L91-L97)）放行 DBCC 后，立即做子命令二次校验，未通过则 Deny：

### 4. 新增 ValidateDbcc 私有方法

```csharp
/// <summary>
/// DBCC 子命令二次校验：仅放行只读诊断白名单子命令，且禁带修复选项。
/// </summary>
private static SqlGuardResult ValidateDbcc(string firstStatement, string normalized)
{
    var tokens = firstStatement.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    if (tokens.Length < 2)
    {
        return SqlGuardResult.Deny("DBCC 缺少子命令", "SQL_BLOCKED");
    }

    // 子命令可能紧贴括号，如 SQLPERF(logspace)
    string sub = tokens[1];
    int paren = sub.IndexOf('(');
    if (paren >= 0) sub = sub[..paren];

    if (!SqlServerDbccReadOnlySubcommands.Contains(sub))
    {
        return SqlGuardResult.Deny(
            $"不允许的 DBCC 子命令: {sub}（仅允许只读诊断子命令）",
            "SQL_BLOCKED");
    }

    // 修复选项检测：CHECKDB/CHECKTABLE/CHECKALLOC 带 REPAIR_* 时写数据
    foreach (string mod in DbccRepairModifiers)
    {
        string pattern = @$"\b{Regex.Escape(mod)}\b";
        if (Regex.IsMatch(normalized, pattern, RegexOptions.IgnoreCase | RegexOptions.CultureInvariant))
        {
            return SqlGuardResult.Deny(
                $"DBCC 修复选项不允许: {mod}",
                "SQL_BLOCKED");
        }
    }

    return SqlGuardResult.Allow();
}
```
